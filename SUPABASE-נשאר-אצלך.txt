סנכרון עם Supabase — מה אתה עושה בפרויקט ב-Supabase (פעם אחת + דומיין)
================================================================

1) יצירת פרויקט ב-Supabase (אם אין): https://supabase.com/dashboard

2) API keys  
   Project Settings → API  
   • Project URL  
   • anon public key  
   להדביק בקובץ supabase-config.js (במחשב שלך, לא בגיט).

3) טבלאות ואבטחה  
   SQL Editor → להדביק ולהריץ את הקובץ supabase-schema.sql מהפרויקט.  
   אם שורת publication נופשת שגיאה — ב-Database → Publications הוסף ידנית את הטבלאות ל-realtime.

4) Google להתחברות  
   Authentication → Providers → Google → הפעלה + Client ID/Secret מ-Google Cloud Console (כמו בפרויקטים האחרים שלך ב-Supabase).

5) כתובות Redirect ל-OAuth  
   Authentication → URL Configuration  
   • Site URL: כתובת האתר שלך (למשל Netlify) או לפחות https://xxxx.netlify.app  
   • Redirect URLs: להוסיף את כתובת האתר המלאה וגם http://localhost:* לבדיקות מקומיות.

6) העלאה ל-Netlify (מומלץ — בלי להעלות מפתחות לגיט)
   חשוב: משתני סביבה ב-Team (משותפים לכל הפרויקטים) הם בתשלום — לא שם.
   בחר את האתר → Site configuration → Environment variables (רמת אתר = בחינם):
      • SUPABASE_URL      — מהדשבורד Supabase → Project Settings → API → Project URL
      • SUPABASE_ANON_KEY — אותו מסך → anon public key
   ב. חיבור לריפו מ-GitHub או deploy ידני — הקובץ netlify.toml מריץ:
      npm run build
      הסקריפט scripts/generate-supabase-config.js יוצר את supabase-config.js רק בזמן בנייה.
   ג. פריסה מקומית ידנית:
      npx netlify-cli deploy --dir . --prod
      (גם שם Netlify יספק את משתני הסביבה אם מוגדרים באתר.)

זכור: supabase-config.js לא נשמר בגיט — במחשב חדש: העתק מ-supabase-config.example.js או הגדר משתני סביבה והרץ npm run build.
